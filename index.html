Bot Setup Guide

Bot Setup Guide

Build a personal AI that runs 24/7 on your own machine. Each section is independent. Check off steps as you go — progress saves automatically.

Hardware
Get a dedicated computer

Your AI needs its own machine that never turns off. The best option is a Mac mini — it's small (about the size of a sandwich), silent, and uses very little electricity ($2-3/month). A spare laptop with the lid closed also works. If you don't have spare hardware, you can rent a cloud server (called a VPS) from a provider like Hetzner or DigitalOcean for about $5-10/month.

This should not be the computer you use every day. Your AI gets its own dedicated machine so it can run 24/7 without interrupting your work.

Install Homebrew

First, open Terminal. Terminal is an app on your Mac that lets you type commands directly to the computer. To open it: press the Command key (⌘) and the Space bar at the same time. A search bar called Spotlight appears in the center of the screen. Type the word terminal and press Enter. A window with a dark or white background and a blinking cursor will open. This is Terminal.

Now copy the line below (click it to copy), go to Terminal, and paste it by pressing Command + V. Then press Enter.

copy/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

It will ask for your Mac password (the one you use to log in). When you type it, nothing will appear on screen — no dots, no stars, nothing. That's completely normal and a security feature. Just type your password and press Enter.

The installation takes 2-5 minutes. You'll see a lot of text scrolling by. Wait until you see a new blinking cursor, which means it's done.

Prevent the computer from sleeping

Your AI can't work if the computer falls asleep. Open Terminal (Command + Space, type "terminal", press Enter) and paste this command:

copysudo pmset -a sleep 0

The word sudo means "do this with administrator permission." It will ask for your password again (same invisible typing as before). This tells your Mac to never go to sleep.

Also go to System Settings → Displays → Advanced and turn off "Automatically adjust brightness." If this is a laptop, go to System Settings → Battery → Options and turn on "Prevent automatic sleeping on power adapter."

Create a Gmail account for the AI

Your AI needs its own email address to sign up for services like GitHub and Vercel. Go to Gmail signup on your phone (Google often requires a phone number for verification, and it's easier to do this from a phone where it can text you automatically).

Pick any name you like — this is a throwaway identity, not something you'll use personally. Once created, open a browser on the AI's computer (the dedicated machine), go to gmail.com, and log in with this new account. Leave it logged in.

Never use your personal Gmail. The AI's accounts are separate from yours on purpose. If something ever goes wrong, the only thing at risk is a throwaway email — not your personal life.
OpenClaw
Install and start OpenClaw

Open Terminal on the AI's computer (press Command + Space, type "terminal", press Enter). Paste this entire block and press Enter:

copybrew install openclaw openclaw gateway install openclaw gateway start

This does three things: installs OpenClaw, registers it as a background service (so it starts automatically when the computer boots), and starts it right now. Once running, it stays running even if you close Terminal or restart the computer.

Connect Telegram

Telegram is a free messaging app (like WhatsApp or iMessage). This is how you'll talk to your AI every day — you send a message on Telegram, and the AI replies in the same chat, just like texting a friend.

If you don't have Telegram, install it on your phone from the App Store or Play Store. Then follow the OpenClaw Telegram setup guide. The process takes about 5 minutes and involves creating a "bot" through Telegram and pasting a token into OpenClaw's settings.

Once connected, send your AI a test message like "Hello, are you there?" It should reply within a few seconds.

Telegram isn't the only option. OpenClaw also works with Signal, Discord, Slack, WhatsApp, and iMessage. But Telegram is the easiest to set up and works on every platform, so we recommend starting there.
Let the AI create its own accounts

Now that you can talk to your AI through Telegram, tell it to sign up for the services it needs. Send it a message like:

"Sign up for GitHub, Vercel, and Supabase using your Gmail account."

The AI will open a browser on its computer, go to each website, click "Sign up with Google," and handle the registration process. You might need to help with CAPTCHAs (those "click all the traffic lights" puzzles) — the AI will tell you when it gets stuck.

If you want the AI to be able to make small purchases (like buying a domain name for $11/year), create a free virtual credit card at Privacy.com with a low spending limit ($20/month is plenty). You type the card number into the browser yourself when the AI asks — it never sees or stores the full card number.

Share your existing projects

If you already have projects on services like GitHub or Vercel, invite the AI so it can work on them with you:

GitHub: Go to your repository → Settings → Collaborators → Add people → type the AI's GitHub username.

Vercel: Go to your team settings → Members → Invite → enter the AI's Gmail address.

Supabase: Go to your organization settings → Members → Invite → enter the AI's Gmail address.

Google Docs: Open the doc → click Share → enter the AI's Gmail address.

You can stop here and have a fully working setup. At this point you have an AI assistant on Telegram that can write code, deploy websites, manage databases, search the web, and research anything you throw at it. Everything below makes the experience better — but nothing below is required.
Obsidian
Install Obsidian on the AI's computer

Obsidian is a free app for reading and writing notes. Your AI already writes everything in Markdown (a simple text format with basic formatting like bold and bullet points). Obsidian takes those plain text files and displays them beautifully — like reading a polished document instead of raw code.

Open Terminal on the AI's computer (Command + Space, type "terminal", press Enter) and paste:

copybrew install --cask obsidian

Once installed, open Obsidian (you can find it in your Applications folder, or just press Command + Space and type "obsidian"). When it opens, click "Open folder as vault." A file picker window appears. Press Command + Shift + G (this lets you type a folder path directly instead of clicking through folders), type ~/.openclaw/workspace, and click Open.

You should now see all of the AI's files in Obsidian's sidebar: its memory logs, notes, configuration files, and anything else it has created.

Enable Obsidian Sync

Obsidian Sync ($4/month) encrypts all your files with a password that only you know and syncs them across all your devices. Not even Obsidian's servers can read your content — everything is end-to-end encrypted.

To set it up: In Obsidian, click the gear icon (⚙️) at the bottom left → click Core plugins in the sidebar → scroll down to find Sync → toggle it on. Now click Sync in the left sidebar (a new icon appeared) → log in with your Obsidian account (create one if needed at obsidian.md/account) → click Create new remote vault and give it any name you like.

Then open the Sync settings and find "Excluded folders." Add Code to this list. Code repositories use Git for version control, which is completely separate from Obsidian Sync. If you don't exclude this folder, thousands of build files would flood your sync and use up all your storage.

Leave "Sync all file types" turned off. The default setting (Markdown files only) is correct. Code projects contain binary files, compiled assets, and dependency folders that would overwhelm your sync storage and cause conflicts.
Keep Obsidian running forever

Obsidian only syncs while the app is open. On the AI's computer, you need Obsidian to launch automatically when the computer starts and relaunch itself if it ever gets closed. You can set this up with a single Terminal command.

Open Terminal (Command + Space, type "terminal", press Enter) and paste this entire block:

copycat > ~/Library/LaunchAgents/md.obsidian.keepalive.plist << 'PLIST' Labelmd.obsidian.keepalive ProgramArguments open-aObsidian RunAtLoad StartInterval300 PLIST launchctl load ~/Library/LaunchAgents/md.obsidian.keepalive.plist

This creates a small configuration file and activates it immediately. From now on, Obsidian will launch every time the computer starts and check every 5 minutes that it's still running. If it somehow gets closed, it reopens itself automatically.

Connect your phone and laptop

Install the Obsidian app on each device you want to use:

Phone: Download from the App Store or Play Store (free).

Laptop/Desktop: Download from obsidian.md/download (free).

On each device: open Obsidian → create a new vault (any name) → go to Settings (gear icon) → Sync → log in with the same Obsidian account you used on the AI's computer → select the remote vault you created earlier.

Wait about a minute for it to download all the files. Once synced, everything the AI writes appears on your phone within seconds. You can also write notes from your phone and the AI will see them.

Enable semantic search

By default, your AI can only search its own daily log files. Semantic search lets it search through all your notes by meaning, not just exact keywords. For example, you could ask "what restaurant did James recommend?" and it would find the answer even if the word "restaurant" never appears in the note — because it understands that "that Italian place on 5th street" is a restaurant.

Everything runs locally on your machine. No data is sent to any external service.

Open Terminal on the AI's computer (Command + Space, type "terminal", press Enter) and paste these two commands:

copyopenclaw config set agents.defaults.memorySearch.provider local openclaw config set agents.defaults.memorySearch.fallback none

Next, you need to tell the AI which folders to search. Open the configuration file by pasting this in Terminal:

copyopen ~/.openclaw/openclaw.json

This opens the file in TextEdit (or your default text editor). Find the memorySearch section and add an extraPaths line listing the folders you want searchable:

"extraPaths": ["Contacts", "Lists", "Memos", "Tasks"]

Replace those folder names with whatever folders you've created in your vault. Save the file (Command + S) and close it.

Finally, restart OpenClaw so it picks up the changes. Paste this in Terminal:

copyopenclaw gateway restart

The first time semantic search runs, it downloads a small AI model (~313MB) and builds an index of all your notes. This takes 2-5 minutes depending on your internet speed and how many notes you have. After that, searches are instant.

This is the most impactful step in this section. Without semantic search, the AI can only look through its own daily logs. With it, the AI can find information across everything you've ever written — contacts, meeting notes, lists, ideas, anything in your vault.
Security
Dedicated machine, separate accounts

The AI should never run on your personal computer, and it should never be logged into your personal accounts. It gets its own machine (or at minimum its own user account on a shared machine), its own Gmail, its own GitHub, its own everything.

The reason is simple: if the AI's accounts were ever compromised, the blast radius should be limited to a throwaway Gmail and a few free-tier developer accounts — not your personal life, your bank, or your work.

Gateway stays local

The OpenClaw gateway (the background service running on the AI's computer) only listens on 127.0.0.1, which is the computer's way of talking to itself. This means only software running on the same machine can communicate with the gateway. No one on the internet can reach it.

Never change this setting. If you need to access the AI's computer remotely (from your laptop or phone), use Tailscale (free for personal use). Tailscale creates a private, encrypted tunnel between your devices without opening any ports to the internet. Install it on both machines, log in with the same account, and they can see each other securely.

Write safety rules

Your AI has a file called SOUL.md in its workspace (you can see it in Obsidian). This file defines the AI's personality and boundaries. Open it and add clear rules at the top, such as:

"Never send emails, make purchases, or post anything publicly without asking me first. Never access files outside the workspace. Prefer reversible actions — use trash instead of permanent delete."

These aren't unbreakable laws (AI safety is an active area of research), but they significantly reduce the chance of the AI doing something you didn't intend. Think of it like setting expectations with a new employee.

Use scoped tokens with expiration dates

When the AI creates API tokens (passwords that let it access services like GitHub or Vercel programmatically), always choose the minimum permissions needed. If the AI only needs to read code, don't give it write access. Set tokens to expire after 90 days so they don't last forever if forgotten.

Your AI can help with this. Send it a message like: "Review your current API tokens and tell me which ones have excessive permissions or no expiration date."

Set up remote screen access

About 5% of the time, the AI will hit something it can't handle on its own: a CAPTCHA puzzle, a credit card form, or a browser security check. When that happens, you need to see its screen.

Mac to Mac (same network): On the AI's computer, go to System Settings → General → Sharing → turn on Screen Sharing. From your Mac, open Finder → in the menu bar click Go → Network. You'll see the AI's computer listed. Double-click it to connect.

From anywhere (different network): Install Tailscale on both machines. Once connected, you can use any VNC app (like the built-in Screen Sharing app on Mac) to connect to the AI's Tailscale IP address. This works from anywhere in the world as if you were on the same local network.

What's still risky and why we do all of the above: Prompt injection — where malicious content on a webpage tricks the AI into doing something unintended — is an unsolved problem across the entire AI industry. That's exactly why the AI uses throwaway accounts with limited permissions. The LLM provider (Anthropic, OpenAI, etc.) processes your conversations on their servers, so avoid sharing truly sensitive information like passwords, financial account numbers, or social security numbers through the AI. Only install OpenClaw skills from sources you trust.

Pro Tips
Keep your folders one level deep. When you create folders in your vault (Contacts, Lists, Memos, Tasks, etc.), resist the urge to nest folders inside folders. Put every note directly inside one top-level folder. If you can't find a note by scanning a flat list of filenames, the fix is to rename the file to something more descriptive — not to bury it three levels deep. Deep folder trees make things invisible. A flat structure keeps everything discoverable. Once you've set up your folder structure, tell the AI how you like things organized (e.g., "contact info goes in Contacts, project notes go in Memos") and it will file things correctly on its own.
Create a Whiteboard.md file at the root of your vault. Since it starts with "W," it naturally sorts to the bottom of your file list in Obsidian's sidebar, making it easy to find no matter how many other files you have. Use it as a shared scratchpad between you and the AI: paste links you want to read later, jot quick thoughts via voice-to-text on your phone, dump ideas you don't want to forget. Because it lives in the vault root, the AI can see everything you put there. Next time you chat, you can say "check my whiteboard" and it knows exactly what you're talking about.
Put all code in a folder called Code/. Every repository the AI clones, every project it creates, every OpenClaw skill it installs — all of it goes in the Code folder. This matters because you excluded Code from Obsidian Sync (back in the Obsidian section). Code is managed by Git (a version control system designed for source code), while your notes are managed by Obsidian Sync. They use completely different sync mechanisms and would conflict badly if mixed together. Keeping them in separate folders means they never interfere with each other. Tell your AI early: "Always clone repos into Code/."
Give it a real task on day one. Don't spend a week perfecting configuration files before doing anything useful. As soon as Telegram is connected and you can chat with the AI, ask it to do something real: "Build me a personal website." "Research the best hotels in Kyoto for April." "Read this PDF and summarize the key points." "Write a Python script that renames all my photos by date." You'll learn how the system works by actually using it, and you'll quickly discover what your AI is great at and where it needs guidance.
Let the AI finish setting itself up. Once OpenClaw and Telegram are working, you can paste this page's URL into your Telegram chat and say "follow this guide to set up Obsidian and semantic search." The AI can read this page and complete many of the remaining steps by itself. This guide is designed to be both human-readable and machine-readable — the AI understands the Terminal commands, file paths, and configuration changes described here.
Run updates weekly. Open Terminal on the AI's computer (Command + Space, type "terminal", press Enter) and paste:
copyopenclaw update

OpenClaw ships new features and security patches frequently. Staying on the latest version means fewer bugs and better capabilities. You can also just ask your AI to update itself — send it "run openclaw update" on Telegram.
