Build Your Own AI Assistant

A personal AI that runs 24/7 on your own machine. It reads your notes, writes code, deploys websites, and replies to your messages. You talk to it through Telegram. You see its work in Obsidian, synced to all your devices. Everything it knows is searchable by meaning, running locally, with zero data sent to anyone.

This guide is step-by-step and copy-pasteable. No technical background required.

1. Get a machine

Your AI needs a computer that stays on 24/7 and is always connected to the internet. This is its home.

Best option: A Mac mini. Small, quiet, energy-efficient, powerful enough for everything.

Other options: A spare laptop (set it to never sleep), a Linux desktop, or a cloud server (VPS) from any provider.

The key is that it never turns off. Your AI lives here. If the machine sleeps, the AI sleeps.

Prevent sleep (Mac): sudo pmset -a sleep 0

Display sleep is fine. The machine just can't fully sleep.

2. Install Homebrew

Homebrew is a tool that lets you install software on a Mac or Linux machine. You need it for the next steps.

How to open Terminal: On a Mac, press Command + Space, type "terminal", and press Enter. A window will open. This is where you type commands.
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

It will ask for your password (the one you use to log into the computer). Type it and press Enter. You won't see the letters as you type. That's normal.

3. Install OpenClaw

OpenClaw is the software that runs your AI assistant. It gives the AI access to tools, memory, files, and your chat apps.

brew install openclaw

Wait a minute or two. When it's done, start the gateway (the background service that keeps your AI running):

openclaw gateway install openclaw gateway start

The gateway runs in the background. It survives reboots. You can close the terminal.

4. Connect Telegram

This is how you'll talk to your AI. Telegram is a free messaging app that works on your phone, tablet, and computer. You message your AI just like texting a friend.

OpenClaw also works with Signal, Discord, Slack, iMessage, and others. But Telegram is the easiest to set up and the most reliable.

Follow OpenClaw's setup guide for connecting Telegram. Once connected, send a message. Your AI will reply in the same chat.

5. Create a Gmail for your AI

Your AI needs its own email address to sign up for services. Create a new Gmail account, but do it on your phone, not the computer.

Why your phone? Google requires a text message verification code. On your phone, you can receive the text and enter the code immediately. On desktop, it's harder.

Once created, open a browser on the AI's machine, go to gmail.com, and log in with the new account. Keep it logged in.

Use a dedicated email. Never use your personal Gmail. This is the AI's identity. If anything goes wrong, the damage is limited to a throwaway Gmail account.

6. Let it create accounts

Now that your AI has an email, tell it (through Telegram) to sign up for services:

GitHub (for code), Vercel (for hosting websites), Supabase (for databases), or anything else you want it to use.

The AI will open a browser, click "Sign up with Google", and authenticate itself. Most flows work automatically.

If you want the AI to buy things (like domain names), give it a virtual credit card from Privacy.com (free). Set a low spending limit ($20/month). You type the card number manually when needed because payment fields block automation.

7. Connect Obsidian

This is the power move. Your AI's workspace is a folder of Markdown files. Obsidian is the best Markdown editor in the world. Connect them and you can see everything your AI writes, from any device, beautifully formatted.

brew install --cask obsidian

Open Obsidian. Click "Open folder as vault." Press Cmd+Shift+G and type:

~/.openclaw/workspace

Click Open. You'll see all the AI's files: its memory, its notes, its daily logs.

Enable Obsidian Sync ($4/month, end-to-end encrypted):

Settings > Core plugins > Toggle Sync on. Log in, create a remote vault. In Sync settings, add Code to Excluded folders. Keep everything else at defaults.

Do not enable "Sync all file types." The default (Markdown only) is correct. Code repos contain thousands of build artifacts that will overwhelm your sync. We learned this the hard way.

Keep Obsidian running forever. Save this as ~/Library/LaunchAgents/md.obsidian.keepalive.plist:

<?xml version="1.0" encoding="UTF-8"?> <!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd"> <plist version="1.0"> <dict> <key>Label</key> <string>md.obsidian.keepalive</string> <key>ProgramArguments</key> <array> <string>open</string> <string>-a</string> <string>Obsidian</string> </array> <key>RunAtLoad</key> <true/> <key>StartInterval</key> <integer>300</integer> </dict> </plist>
launchctl load ~/Library/LaunchAgents/md.obsidian.keepalive.plist

This launches Obsidian on boot and checks every 5 minutes that it's still running.

Connect from your other devices: Install Obsidian on your phone or laptop. Create a new vault, enable Sync, log in, select the remote vault. That's it. No extra configuration needed on secondary devices.

8. Enable semantic search

This makes every note in your vault searchable by meaning, not just keywords. Ask your AI "what did we discuss about pricing?" and it finds the answer across hundreds of notes, even if the word "pricing" never appears. Everything runs locally. Zero data sent to anyone.

openclaw config set agents.defaults.memorySearch.provider local openclaw config set agents.defaults.memorySearch.fallback none

Then edit ~/.openclaw/openclaw.json and add your folder names so they get indexed. Under agents.defaults.memorySearch, add:

"extraPaths": ["Contacts", "Lists", "Memos", "Tasks"]

Replace those with whatever folders you create. Restart the gateway. The first run downloads a small model (~313MB) and indexes everything. After that, new files are indexed automatically.

This is the step most people skip. Without it, your AI only searches its own memory folder. With it, the AI searches across your entire vault by semantic meaning. It's the difference between a search engine and a second brain.

9. Lock it down

You're giving an AI access to real accounts and real tools. Here's how to keep the blast radius small.

Dedicated machine. Never run OpenClaw on your daily-use computer. Use a Mac mini, spare laptop, or cloud server. The AI gets its own desk. If it breaks something, your personal files are untouched.

Separate everything. The AI gets its own Gmail, GitHub, Vercel, and any other accounts. Never share yours. Use Privacy.com for a virtual credit card with a $20 limit.

Gateway stays local. The OpenClaw gateway should only listen on 127.0.0.1 (loopback). Never expose it to the internet. Use Tailscale (free) if you need remote access.

SOUL.md is the AI's constitution. Put safety rules at the top: no reading outside the workspace, no running destructive commands, no sending emails without approval, no spending money without approval. It's guidance, not enforcement, but it works.

Scoped tokens. Every API token should have the minimum permissions needed. Read-only where possible. Short expiration where possible. If the AI's entire identity were compromised, the worst outcome should be losing a throwaway Gmail and a $20 virtual card.

What's still risky. Prompt injection is an unsolved problem. A malicious email to the AI's Gmail could trick it. That's why it gets a throwaway email. The LLM provider (Anthropic, OpenAI, etc.) sees your prompts and responses. Don't feed it truly secret information. Skills can be malicious. Only use skills you've reviewed.

10. Set up remote access

Sometimes the AI can't do something on its own: CAPTCHAs, payment card fields, or browser protections. When that happens, you need to see its screen and type for it. This is rare (maybe 5% of the time).

Easiest (Mac to Mac): Turn on Screen Sharing in System Preferences. From your Mac, open Finder > Go > Network, find the AI's computer, and click Screen Sharing.

From anywhere: Install Tailscale on both machines (free, encrypted). Connect via VNC over Tailscale. No ports exposed to the internet.

11. Share your projects

Invite your AI to your stuff:

GitHub: Add its username as a collaborator.
Vercel: Invite its email to your team.
Supabase: Add it to your organization.
Google Docs: Share with its Gmail.

Now it can push code, deploy websites, query databases, and edit documents alongside you.

The end result: You message your AI through Telegram. It reads, thinks, and acts. Everything it writes appears in Obsidian on your phone. Every note you write in Obsidian is searchable by your AI in milliseconds. You have a collaborator that never sleeps, never forgets, and keeps getting better.

12. Pro tips

None of these are required. They're things we've found useful after running this setup daily.

Pro Tip

Keep folders one level deep. Contacts, Lists, Memos, Tasks, whatever makes sense to you. Don't nest three levels deep because nobody ever finds anything in there. Tell your agent how you want things organized and it will sort your notes into the right folders for you.

Pro Tip

Create a Whiteboard.md file. Put it at the root of your vault. "W" sorts near the bottom, so it's always easy to find. Use it as a scratchpad: quick thoughts, voice note dumps, links to check later. Think of it as your vault's homepage.

Pro Tip

All code goes in a Code/ folder. Git repos, cloned projects, skills. Exclude it from Obsidian Sync (you already did this in step 7). Git handles code sync. Obsidian handles notes. They never overlap.

Pro Tip

Update regularly. Run openclaw update and openclaw security audit --deep weekly. New features and security patches ship often.

Pro Tip

Start with a task. Don't configure for a week before doing anything useful. Set up steps 1-4, then immediately ask your AI to do something real. Build a website. Research a topic. Organize your files. You'll learn the system by using it.
