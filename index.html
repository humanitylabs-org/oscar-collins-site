Build Your Own AI Assistant

A free, step-by-step guide. Three steps from zero to a personal AI that runs 24/7 on your own hardware.

What You're Building

A personal AI assistant that runs on your own machine, accessible from anywhere, with all its files synced to your devices. It can write code, deploy websites, manage databases, search the web, and remember everything.

Three steps. Each one unlocks a new capability. You can stop after any step and have something useful.

1

Install OpenClaw

Set up a dedicated machine (Mac mini, spare laptop, or VPS) running your AI 24/7 with remote access via Tailscale.

~30 minutes
2

Start Chatting

Connect Telegram and start talking to your AI from your phone. Give it accounts, invite it to your projects.

~15 minutes
3

Sync Your Files

See everything your AI writes on all your devices. Chat from inside Obsidian. Search notes by meaning.

~20 minutes
What you'll need: A spare Mac mini, old laptop, or $5/month cloud server. A phone with Telegram. About an hour. Everything else is free.
Already have a bot? Skip ahead to whichever step you need. Check off steps you've already completed -- progress saves automatically. You can also send your bot this page and let it finish the setup for you.
Step 1
Install OpenClaw
Goal: A dedicated machine running your AI 24/7 with remote access from anywhere.
Pick a machine

Your AI needs its own machine that never turns off:

Mac mini or spare laptop (recommended) -- small, silent, low power ($2-3/month electricity). A spare laptop with the lid closed works too. This should not be the computer you use every day.

VPS -- rent a cloud server from Hetzner or DigitalOcean for $5-10/month. Good if you don't have spare hardware.

Use a dedicated machine. Your AI gets its own computer, its own Gmail, its own accounts. If something goes wrong, only throwaway accounts are at risk -- not your personal life.
Open a terminal

Mac: Press Command (⌘) + Space, type terminal, press Enter. A window with a blinking cursor opens -- this is where you type commands.

VPS: From your personal computer's terminal:

copyssh root@your-vps-ip
Install prerequisites

Mac: Install Homebrew (a package manager):

copy/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

It asks for your Mac password. When you type it, nothing appears on screen -- no dots, no stars. That's normal. Wait 2-5 minutes.

VPS (Ubuntu/Debian):

copyapt update && apt upgrade -y
Prevent sleeping (Mac only)
copysudo pmset -a sleep 0

Also: System Settings → Displays → Advanced → turn off "Automatically adjust brightness." For laptops: Battery → Options → prevent automatic sleeping on power adapter.

VPS users: Skip this. Cloud servers don't sleep.
Create a Gmail for the AI

Go to Gmail signup on your phone (easier for verification). Pick any name. This is the AI's email for signing up to services.

Then on the AI's machine, open a browser, log in with the new account, and leave it logged in.

Install OpenClaw
copycurl -fsSL https://openclaw.ai/install.sh | bash

This installs OpenClaw, registers it as a background service (auto-starts on boot), and starts it immediately.

Mac alternative: brew install openclaw then openclaw gateway install && openclaw gateway start

Install Tailscale

Tailscale (free) creates a private encrypted network between all your devices, anywhere in the world. You'll use it for remote access and file syncing.

Mac:

copybrew install --cask tailscale

Open Tailscale from Applications. Click the menu bar icon and sign in. Note the Tailscale IP (like 100.x.x.x).

VPS (Linux):

copycurl -fsSL https://tailscale.com/install.sh | sh sudo tailscale up

Follow the auth URL. Note the Tailscale IP.

Install Tailscale on your phone (App Store / Play Store) and laptop (tailscale.com/download). Sign in with the same account.

VPS security: Lock down your firewall so only Tailscale can reach OpenClaw:
ufw default deny incoming && ufw allow ssh && ufw allow in on tailscale0 && ufw enable
Enable the dashboard
copyopenclaw config set gateway.controlUI.enabled true openclaw gateway restart

From any device on Tailscale, open http://[Tailscale IP]:18789. Bookmark it on your phone.

Mac screen sharing: System Settings → General → Sharing → turn on Screen Sharing. Access from another Mac: Finder → Go → Connect to Server → vnc://[Tailscale IP]. Useful when the AI hits CAPTCHAs.
Step 1 complete. Your AI is running 24/7 with remote access. Next: talk to it.
Step 2
Start Chatting
Goal: Talk to your AI from your phone via Telegram, like texting a friend.
Set up Telegram

Install Telegram on your phone from the App Store or Play Store.

Then follow the OpenClaw Telegram guide (~5 minutes). You create a "bot" in Telegram and paste a token into OpenClaw.

Other options: OpenClaw also works with Signal, Discord, Slack, WhatsApp, and iMessage. Telegram is the easiest to start with.
Send your first message

Open Telegram and message your AI: "Hello, are you there?"

It should reply within seconds. If not, check the dashboard for errors. Common fix: openclaw gateway restart

Give it a real task right away. "Build me a personal website." "Research hotels in Kyoto for April." "Summarize this PDF." You learn by using it, not by tweaking.
Let the AI create its own accounts

Tell it: "Sign up for GitHub, Vercel, and Supabase using your Gmail account."

It opens a browser, goes to each site, and handles registration. Help with CAPTCHAs via screen sharing (Mac) or paste verification codes (VPS).

For small purchases (like a $11/year domain), create a free virtual card at Privacy.com with a $20/month limit.

Use scoped tokens. When the AI creates API keys, make sure they have minimum permissions and expire after 90 days. Ask it: "Review your tokens and flag any with excessive permissions."
Invite it to your projects

GitHub: Repository → Settings → Collaborators → Add the AI's username.

Vercel: Team → Members → Invite the AI's Gmail.

Supabase: Organization → Members → Invite the AI's Gmail.

Google Docs: Share → Enter the AI's Gmail.

Set safety boundaries

Open SOUL.md in the AI's workspace (you'll see it in the dashboard or via terminal) and add rules:

"Never send emails, make purchases, or post publicly without asking me first. Never access files outside the workspace. Use trash instead of permanent delete."

Like expectations for a new employee. Not unbreakable, but significantly reduces unintended actions.

Step 2 complete. You have a fully working AI assistant. It can write code, deploy websites, manage databases, and research anything. Everything after this is optional upgrades.
Your bot can finish the rest. Send it: "Read botsetupguide.com and set up file syncing for me. Ask me questions if you need input." Step 3 is written so your AI can read and execute it directly.
Step 3
Sync Your Files
Goal: See everything your AI writes on all your devices -- notes, memories, files -- beautifully rendered and updated in real time.
Choose your sync method:

Already use Obsidian Sync? Keep it. Follow Path A. It also syncs plugin settings across devices automatically.

Starting fresh, or on a VPS? Use Syncthing (free, peer-to-peer). Follow Path B. This is the default recommendation.

VPS users must use Path B. Obsidian Sync requires the desktop app with a GUI.
Path A
Obsidian Sync (if you already have it)
You already pay for Obsidian Sync. Keep using it. Skip Path B.
A: Install Obsidian on the AI's Mac
copybrew install --cask obsidian

Open Obsidian. Click "Open folder as vault." Press Command + Shift + G, type ~/.openclaw/workspace, click Open.

A: Enable Sync

Gear icon → Core plugins → toggle on Sync → click Sync → log in → Create new remote vault.

In Sync settings, add Code to "Excluded folders." Code repos use Git, not Sync.

Leave "Sync all file types" off. Default (Markdown only) is correct.
A: Keep Obsidian running

Obsidian only syncs while open. Auto-launch it:

copycat > ~/Library/LaunchAgents/md.obsidian.keepalive.plist << 'PLIST' Labelmd.obsidian.keepalive ProgramArguments open-aObsidian RunAtLoad StartInterval300 PLIST launchctl load ~/Library/LaunchAgents/md.obsidian.keepalive.plist
A: Connect your devices

Install Obsidian on your phone (App Store / Play Store) and laptop (obsidian.md/download).

On each: new vault → Settings → Sync → same account → select remote vault. Everything appears within seconds.

Path B — Recommended
Syncthing over Tailscale (free, trustless)
Peer-to-peer file sync. No server, no subscription, no one sees your data. Works with everything.
B: Install Syncthing on the AI's machine

Mac:

copybrew install syncthing brew services start syncthing

VPS (Ubuntu/Debian):

copyapt install -y syncthing cat > /etc/systemd/system/syncthing.service << 'EOF' [Unit] Description=Syncthing After=network.target [Service] ExecStart=/usr/bin/syncthing serve --no-browser --no-restart Restart=on-failure User=root [Install] WantedBy=multi-user.target EOF systemctl enable syncthing && systemctl start syncthing

Web UI: http://[Tailscale IP]:8384

VPS: Set a username and password in Syncthing Settings → GUI before accessing over Tailscale.
B: Install Syncthing on your devices

Mac/PC: syncthing.net/downloads or brew install syncthing

iPhone: Mobius Sync ($6 one-time)

Android: Syncthing (free)

B: Pair and share the workspace

On the AI's Syncthing UI: note the Device ID under Actions → Show ID.

On your device: Add Remote Device → paste the ID. Accept on both sides.

Then on the AI's machine: Add Folder → path ~/.openclaw/workspace (or /root/.openclaw/workspace on VPS) → Share with your device.

Accept the folder on your device. Set the local path to your Obsidian vault location.

Ignore patterns: In folder settings, add to Ignore Patterns:
Code
.obsidian
node_modules
B: Open in Obsidian

Install Obsidian (free) on your phone and laptop. Open the synced folder as a vault. Your AI's notes update in real time.

Both Paths
Chat Plugin + Semantic Search
These steps apply no matter which sync method you chose.
Install the OpenClaw plugin

In Obsidian: Settings → Community Plugins → Browse → search "OpenClaw" → Install → Enable.

If the plugin isn't listed yet, install manually:

copycd ~/.openclaw/workspace/.obsidian/plugins mkdir -p openclaw && cd openclaw curl -L -o main.js https://github.com/oscarhenrycollins/obsidianclaw/releases/latest/download/main.js curl -L -o manifest.json https://github.com/oscarhenrycollins/obsidianclaw/releases/latest/download/manifest.json curl -L -o styles.css https://github.com/oscarhenrycollins/obsidianclaw/releases/latest/download/styles.css

Then: Settings → Community Plugins → reload → enable "OpenClaw."

Connect the plugin

The setup wizard appears on first launch. Enter your gateway URL:

Same machine: ws://127.0.0.1:18789

Over Tailscale: ws://[Tailscale IP]:18789

First connection triggers device pairing. On the AI's machine:

copyopenclaw devices list openclaw devices approve <requestId>
Three layers of security. Tailscale encrypts the network. The auth token verifies identity. Ed25519 device fingerprinting prevents unauthorized devices.
Enable semantic search

Search notes by meaning, not keywords. "What restaurant did James recommend?" works even if "restaurant" never appears. Runs locally -- nothing sent externally.

copyopenclaw config set agents.defaults.memorySearch.provider local openclaw config set agents.defaults.memorySearch.fallback none

Open the config and add folders to search:

copyopen ~/.openclaw/openclaw.json
"extraPaths": ["Contacts", "Lists", "Memos", "Tasks"]
copyopenclaw gateway restart

First run downloads a model (~313MB) and indexes notes. After that, instant.

Keep OpenClaw updated
copyopenclaw update

Run weekly, or tell the AI: "Run openclaw update."

Put all code in Code/. Every repo, project, and skill. Excluded from syncing. Code uses Git, notes use sync. Tell the AI: "Always clone into Code/."
Create a Whiteboard.md. Shared scratchpad at the vault root. Paste links, jot thoughts, dump ideas. Say "check my whiteboard" and the AI knows what you mean.
All done. You have a personal AI running 24/7 on your own hardware, accessible from anywhere, synced to all your devices. Welcome to the future.
What's still risky: Prompt injection (malicious content tricking the AI) is unsolved industry-wide -- throwaway accounts matter. The LLM provider sees conversations, so don't share passwords. Only install skills from trusted sources.
